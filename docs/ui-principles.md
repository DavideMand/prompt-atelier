# UI Principles

Prompt Atelier is a working prompt bench, not a marketing site.

## Product Shape

- Use a minimal two-column layout: guided input on the left, generated output on the right.
- Keep the first screen useful immediately.
- Use progressive disclosure so beginners are not buried, but advanced users can tune style, lighting, technical details, and exclusions.
- Make the output easy to inspect, copy, and export.

## Interaction

- Uploaded reference images are for browser-side analysis notes only.
- Never send uploads to a server.
- Never imply that the app generates images.
- Update outputs instantly as form values change.
- Show a quality score with actionable issue hints.

## Visual Tone

- Quiet, studio-like interface.
- Dense enough for production work.
- Clear section boundaries without nested card clutter.
- Typography should feel precise and editorial.
- Use restrained color with meaningful accents.

## Accessibility

- Labels must be explicit.
- Controls must remain keyboard accessible.
- Textareas need enough height for real prompt writing.
- Buttons should name the exact action: copy, export text, export JSON.
- Color must not be the only way to convey state.
